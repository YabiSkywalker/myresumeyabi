What is CI/CD?

CI/CD tends to be overcomplicated when in reality it is a simple concept. Developers are constantly working on the same code, and all need to merge their changes into one centralized location without cause merge conflicts, and that is essentially what the CI part of CI/CD resolves. It allows developers to be able to merge only their latest changes so long as it does not conflict with the existing code in a negative fashion.

Typically, this is done on GitHub or GitLab. This way, many developers can work on the same code, and all their changes can be merged together while addressing merge conflicts.

Continuous Deployment

Continuous Integration (CI) is the process of merging code into one main (or Master) repository. Typically, this is done on GitHub or GitLab. This way, many developers can work on the same code, and all their changes can be merged together while addressing merge conflicts.
